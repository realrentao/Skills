# 生成配音稿

将教学课程逐字稿（.md）转换为纯口述配音稿（.txt），一句一行，可直接用于录音。

## 触发场景

当用户提到以下意图时使用：
- "生成配音稿"
- "提取口述部分"
- "只保留纯口述部分"
- "一句一行"
- "方便做配音"
- 类似表述，表明需要从逐字稿中提取可朗读文本

## 工作流程

### 1. 读取逐字稿文件

读取用户指定的 Markdown 文件路径。

### 2. 识别并移除的非口述元素

以下内容必须完全移除，不保留在输出中：

| 元素类型 | 特征 | 示例 |
|---------|------|------|
| 镜头/拍摄指示 | 以 `> 【` 开头或以 `> **【` 开头 | `> 【镜头：面对镜头...】` |
| 时间戳标记 | 包含 ⏱️ 字样的行 | `## 【开场】⏱️ 约1分钟` |
| 章节标题标记 | Markdown 标题 `#` `##` 行，以及 `---` 分隔线 | `## 【环节一：...】` |
| 表格结构 | Markdown 表格行（含 `\|` 分隔符）以及表头/分割行 | `\| 意大利语 \| 中文 \|` |
| 格式标记 | Markdown 粗体 `**`、引用 `>` 等纯格式符号（保留内容） | `> **"..."**` → 提取内部文本 |
| 脚注/版本信息 | 文件末尾的版本、日期、元数据 | `📝 逐字稿版本：`、`📅 更新时间：` |
| 注释性标注 | 独立的 "读作：" / "中文意思是：" / "翻译：" 等前缀词 | 这些本身是口述内容，不要移除 — 只移除结构化注释行 |

**注意**：下列内容看似注释，但实际上是口述教学的一部分，必须保留：
- "读作：普龙托" — 老师会在录音中说出
- "中文意思是：喂" — 同上
- "翻译：周四下午" — 同上

判断标准：如果老师在录音时会自然说出这句话，就保留；如果是给录制者看的幕后指示，就移除。

### 3. 保留的纯口述内容

以下内容全部保留：

- 意大利语句子和短语（如 `Pronto`、`Buongiorno! Per quando?`）
- 中文翻译和讲解（如 "这是意大利语打电话时的'喂'"）
- 教学口令和口诀（如 "涛子口诀：约时间说 Per，几点用 Alle"）
- 词汇跟读部分（读作 / 中文意思是 / 翻译等）
- 复习和总结部分
- 开头和结尾的问候语

### 4. 格式规则

- **逐句排列**：每个完整的句子/短语独立一行
- **逗号/冒号连接的句子**：如果两个短句紧密相连（如 "好，我们开始！"），保留为一行；如果逻辑上可拆分，则拆分（如 "Per = 对于。quando = 什么时候。" → 分两行）
- **意中双语在同一句**：保持原样，如 `Ciao a tutti! 大家好！`
- **长句含列表**：如 "今天我们学了：6个核心词汇，一个完整的...5句必背的话" — 保留为一行
- 意大利语中的 `?` 和 `!` 保持原样
- 文件编码：UTF-8

### 5. 输出

- 输出文件命名：`[原文件名前缀]_配音稿.txt`
- 与输入文件放在同一目录下
- 直接告知用户文件路径

## 示例

输入（逐字稿.md）：
```
## 【环节一：生词学习】⏱️ 约7分钟

> 【镜头：对准词汇表】

第一个词：Pronto
读作：普龙托
中文意思是：喂

| 意大利语 | 中文 |
|:---|:---|
| Pronto | 喂（电话用） |
```

输出（配音稿.txt）：
```
第一个词：Pronto。
读作：普龙托。
中文意思是：喂。
```
