# SkillHub 每日简报格式模板

> 本模板用于规范每日简报的输出格式，确保IMA知识库中的简报风格统一、信息完整。

## 文件头

```markdown
# SkillHub 每日简报 | YYYY-MM-DD

基于您的记忆痛点（[痛点1]、[痛点2]、[痛点3]...）个性化推荐
```

**要求**：
- 标题必须包含完整日期
- 第二行列出用户的核心痛点（3-6个）
- 痛点从MEMORY.md和近期日志中提取

---

## 今日推荐（为您精选）

### 1. [技能名称] — [一句话价值描述]

**匹配痛点**：[对应用户痛点]

[详细推荐理由，100-150字。必须包含：]
- [功能描述：这个技能是做什么的]
- [解决痛点：如何帮助用户解决具体问题]
- [使用场景：在用户工作流中的具体应用场景]

**核心能力**：[能力1] → [能力2] → [能力3]

**安装**：`skillhub install [技能ID]`

**要求**：
- 推荐3-5个技能
- 每个技能必须匹配一个具体痛点
- 推荐理由必须具体，不能泛泛而谈
- 安装命令必须可执行

---

## 被埋没的金子 Top 5

```markdown
|排名|技能名称|收藏率|埋没原因|爆发潜力|
|---|---|---|---|---|
|#[排名]|[技能名称]|[收藏率]%|[为什么排名低但值得关注的具体原因]|[高/中/低]|
```

**要求**：
- 表格必须包含所有5列
- 排名使用#前缀（如#70）
- 收藏率保留2位小数
- 埋没原因必须具体（如"分类竞争激烈"、"功能描述过于简单"）
- 爆发潜力：高/中/低三档

---

## 全站潜力 Top 10

```markdown
|排名|技能名称|分类|收藏率|潜力分|推荐理由|
|---|---|---|---|---|---|
|[1-10]|[名称]|[分类]|[收藏率]%|[分数]|[一句话推荐理由]|
```

**要求**：
- 表格必须包含所有6列
- 排名1-10
- 潜力分保留2位小数
- 推荐理由控制在20字以内

---

## 分类速览

```markdown
[分类1]（Top100内X个，全站Y个，平均收藏率Z%）：[关键发现1-2句]
[分类2]（Top100内X个，全站Y个，平均收藏率Z%）：[关键发现1-2句]
...
```

**分类列表**（7大分类）：
1. AI 智能
2. 开发工具
3. 效率提升
4. 数据分析
5. 内容创作
6. 通讯协作
7. 安全合规

**要求**：
- 每个分类1-2句话
- 包含Top100数量、全站数量、平均收藏率
- 突出该分类最值得关注的技能

---

## 页脚

```markdown
简报生成时间：YYYY-MM-DD HH:MM
数据来源：SkillHub 全站 Top100 + 7大分类各Top20（共240个技能扫描）
个性化匹配：基于您的 [N] 个记忆痛点 + [M] 个近期任务
```

**要求**：
- 生成时间精确到分钟
- 数据来源说明扫描范围
- 个性化匹配说明基于多少个痛点和任务

## 微信摘要格式

```
📊 SkillHub Daily Brief | YYYY-MM-DD

基于您的[N]个痛点，今日精选[X]个技能：

🎯 [技能1] — [一句话价值]
解决：[痛点描述]

📈 [技能2] — [一句话价值]
解决：[痛点描述]

💎 被埋没的金子：[技能名]（收藏率[X]%，排名#[N]）

📊 今日扫描：240个技能 | 7大分类 | [X]个潜力技能识别

详情见IMA"简报"知识库
```

**要求**：
- 150字以内
- 纯文本，无表格
- 包含emoji增加可读性
- 引导用户查看IMA完整简报
